Leçon 07 · CSS

Introduction CSS

Le CSS — Cascading Style Sheets — est le langage qui habille le HTML. Si HTML est le squelette, CSS est la peau, les vêtements et la mise en scène. On commence ici par comprendre comment il fonctionne avant de le pratiquer.

Anatomie d'une règle CSS

Une règle CSS se compose toujours de la même façon : un sélecteur qui cible un élément HTML, et des déclarations entre accolades qui définissent le style à appliquer. 

/* Sélecteur { propriété: valeur; } */

h1 {
  color: orange;
  font-size: 2rem;
}

p {
  color: #555555;
  line-height: 1.6;
}
sélecteur h1 → cible tous les <h1>
propriété color → quelle chose on change
valeur orange → comment on la change

Les 3 façons d'écrire du CSS

Il existe trois endroits où écrire du CSS. La troisième méthode — le fichier externe — est la bonne pratique à adopter dès le début. 

<!-- 1. Inline : dans l'attribut style -->
<p style="color: red;">Texte rouge</p>

<!-- 2. Interne : dans une balise <style> -->
<style>
  p { color: red; }
</style>

<!-- 3. ✅ Externe : fichier séparé (meilleure pratique) -->
<link
  rel="stylesheet"
  href="style.css">
❌ Inline — difficile à maintenir, pas réutilisable
⚠ Interne — mélange HTML et CSS dans le même fichier
✅ Externe — un fichier CSS pour tout le site, propre et réutilisable

Les sélecteurs de base

Un sélecteur désigne qui va recevoir le style. Il en existe trois essentiels à maîtriser pour commencer. 

/* Sélecteur de balise : cible TOUTES les balises h2 */
h2 {
  color: navy;
}

/* Sélecteur de classe : cible class="btn" */
.btn {
  background: orange;
  padding: 10px 20px;
}

/* Sélecteur d'id : cible id="hero" (unique) */
#hero {
  background: #f0f0f0;
}

La "cascade" — pourquoi CSS s'appelle ainsi

CSS signifie Cascading Style Sheets. La cascade désigne les règles de priorité quand plusieurs styles s'appliquent au même élément. L'ordre est : id > classe > balise. En cas d'égalité, le style écrit en dernier gagne. 

/* 1. Balise : priorité faible */
p {
  color: gray;
}

/* 2. Classe : priorité moyenne */
.important {
  color: orange;
}

/* 3. Id : priorité forte */
#unique {
  color: red;
}

<!-- Quel style s'applique ? -->
<p class="important" id="unique">
  Ce texte sera rouge.
</p>

← Balise p (gris) — priorité faible

← Classe .important (orange) — priorité moyenne

← Id #unique (rouge) — priorité forte ✓

L'id gagne car il a la plus haute spécificité.

Points clés à retenir

  • Une règle CSS = un sélecteur + des déclarations entre { }
  • Toujours utiliser un fichier CSS externe — jamais de style inline en production
  • balise cible tous les éléments, .classe cible un groupe, #id cible un seul élément
  • La priorité : #id > .classe > balise
  • Un commentaire CSS s'écrit /* entre ces symboles */
