Leçon 03 · HTML

Les listes

Les listes sont partout sur le web : menus de navigation, ingrédients, étapes d'un tutoriel... HTML propose deux types selon que l'ordre des éléments a de l'importance ou non.

La liste non ordonnée <ul>

<ul> signifie unordered list. On l'utilise quand l'ordre des éléments n'a pas d'importance. Chaque élément est encadré par une balise <li> (list item). Le navigateur affiche des puces par défaut. 

<ul>
  <li>HTML</li>
  <li>CSS</li>
  <li>JavaScript</li>
</ul>
  • HTML
  • CSS
  • JavaScript

La liste ordonnée <ol>

<ol> signifie ordered list. On l'utilise quand l'ordre des éléments est important : des étapes, un classement, une procédure. Le navigateur numérote automatiquement les éléments. 

<ol>
  <li>Ouvrir le fichier HTML</li>
  <li>Écrire le code</li>
  <li>Sauvegarder</li>
  <li>Ouvrir dans le navigateur</li>
</ol>
  1. Ouvrir le fichier HTML
  2. Écrire le code
  3. Sauvegarder
  4. Ouvrir dans le navigateur

Les listes imbriquées

On peut placer une liste à l'intérieur d'un <li> pour créer des sous-niveaux. C'est utile pour les menus complexes ou les plans de documents. 

<ul>
  <li>Langages web
    <ul>
      <li>HTML</li>
      <li>CSS</li>
    </ul>
  </li>
  <li>Outils
    <ul>
      <li>VS Code</li>
      <li>Git</li>
    </ul>
  </li>
</ul>
  • Langages web
    • HTML
    • CSS
  • Outils
    • VS Code
    • Git

Cas concret : la navigation

Les menus de navigation sont construits avec des listes. Le CSS s'occupera de les afficher horizontalement. C'est exactement comme ça qu'est fait le menu de ce cours ! 

<nav>
  <ul>
    <li><a href="#">Accueil</a></li>
    <li><a href="#">Services</a></li>
    <li><a href="#">Contact</a></li>
  </ul>
</nav>
  • Accueil
  • Services
  • Contact

Points clés à retenir

  • <ul> pour les listes sans ordre (puces), <ol> pour les listes ordonnées (numéros)
  • Chaque élément d'une liste est toujours dans un <li>
  • On peut imbriquer des listes en plaçant un <ul> à l'intérieur d'un <li>
  • Les menus de navigation sont des listes <ul> + <li> + <a>
